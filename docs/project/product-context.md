# Contexto de Produto

Consulte este documento quando precisar de contexto de negócio, público ou intenção do produto.

## Público

O projeto é voltado para desenvolvedores que desejam um ponto de partida sólido para construir aplicações SaaS. Ele serve como referência de arquitetura e boilerplate funcional com autenticação, gerenciamento de usuários e estrutura multi-tenant.

## Princípios de produto

- Demonstrar padrões comuns de aplicações SaaS (multi-tenancy, auth, RBAC).
- Manter a base simples e extensível para diferentes domínios de negócio.
- Tornar a documentação útil para humanos e agentes de IA.
- Preferir convenções do framework a abstrações customizadas.
- Fornecer componentes de UI reutilizáveis e acessíveis.
